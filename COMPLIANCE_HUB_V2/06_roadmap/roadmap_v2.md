# Roadmap V2

## Fase 0: estabilizacao estrutural minima

Objetivo: garantir que a V1 atual seja confiavel antes da V2.

Entregaveis:

- Corrigir publicacao/relatorio/snapshot.
- Unificar contratos client-safe.
- Garantir paridade de report builders.
- Testes do fluxo `cases -> publicResult -> clientCases -> publicReports`.
- Scripts de repair/backfill seguros.

Impacto: alto.

Risco: baixo/medio.

Esforco: 1 a 3 semanas.

## Fase 1: fundacao BigDataCorp-first

Objetivo: separar provider, raw payload, normalizacao e evidencias.

Entregaveis:

- Provider contract.
- Raw snapshots.
- Provider records.
- Normalizers versionados.
- Evidence items.
- Request ledger.
- Cache/rate limit.

Impacto: muito alto.

Risco: medio.

Esforco: 3 a 6 semanas.

## Fase 2: nucleo investigativo

Objetivo: criar dossie PF/PJ reutilizavel.

Entregaveis:

- Subject/Person/Company.
- Relationships.
- Lawsuits/warrants.
- Timeline.
- Evidence viewer.
- Busca global.
- Dossie no portal ops.

Impacto: muito alto.

Risco: medio.

Esforco: 4 a 8 semanas.

## Fase 3: decisao e sinais

Objetivo: tornar score/veredito explicavel.

Entregaveis:

- RiskSignal.
- Decision.
- Decision reasons.
- Analyst review.
- Report snapshot.
- Explainability.

Impacto: alto.

Risco: medio.

Esforco: 3 a 6 semanas.

## Fase 4: diferenciais competitivos

Objetivo: ampliar produto para plataforma investigativa completa.

Entregaveis:

- Grafo visual.
- Watchlists.
- Monitoramento continuo.
- Alertas.
- Templates por segmento.
- BI operacional.
- API comercial.

Impacto: alto.

Risco: alto.

Esforco: 8 a 16 semanas.
