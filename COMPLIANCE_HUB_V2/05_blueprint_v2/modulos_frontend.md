# Modulos frontend propostos

## Portal cliente

Objetivo: simples, seguro e comercial.

Modulos:

- Dashboard.
- Solicitacoes.
- Nova solicitacao.
- Pendencias/correcao.
- Relatorios.
- Auditoria visivel.
- Equipe/permissoes.

Nao deve expor:

- Payload bruto.
- Nome de APIs.
- Metodologia interna.
- Evidencias sensiveis nao aprovadas.

## Cockpit operacional

Objetivo: produtividade do analista.

Modulos:

- Fila inteligente.
- Caso.
- Dossie da entidade.
- Evidencias.
- Timeline.
- Relacoes.
- Sinais de risco.
- Revisao.
- Decisao.
- Previa do relatorio.
- Auditoria do caso.

## Dossie investigativo

Seccoes:

- Identificacao.
- Documentos.
- Enderecos.
- Telefones/e-mails.
- Empresas relacionadas.
- Socios/administradores.
- Processos.
- Mandados.
- Sanctions/PEP.
- Midia negativa.
- Evidencias.
- Historico de consultas.

## Report composer

Objetivo: relatorio consistente com decisao revisada.

Recursos:

- Snapshot usado.
- Secoes habilitadas.
- Evidencias permitidas.
- Texto executivo.
- Parecer final.
- Proximos passos.
- Link publico.

## Admin

Modulos:

- Clientes/tenants.
- Planos e modulos.
- Providers e quotas.
- Auditoria.
- Saude de APIs.
- Templates.
