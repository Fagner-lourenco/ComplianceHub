# Decisao final

## A V2 deve se parecer com Marble, Ballerine, hibrido ou arquitetura propria?

Decisao: arquitetura propria inspirada parcialmente nos dois.

Marble deve inspirar:

- Case management.
- Decisioning.
- Screening.
- Data model/pivots.
- Continuous screening futuro.

Ballerine deve inspirar:

- Workflow.
- Collection flow.
- Manual review.
- Documentos.
- Backoffice por entidade.

ComplianceHub deve criar do zero:

- BigDataCorp-first architecture.
- Modelo brasileiro PF/PJ/processos/mandados.
- Evidence/provenance store.
- Dossie investigativo.
- Relatorio comercial auditavel.

## O que entra obrigatoriamente na V2

- Provider contract.
- Raw snapshots.
- Entidades canonicas.
- Evidence store.
- Risk signals.
- Decision explicavel.
- Report snapshot imutavel.
- Cockpit investigativo inicial.
- Portal cliente seguro.

## O que nao deve entrar ainda

- Rule builder visual completo.
- SDK publico.
- Marketplace de providers.
- Monitoramento continuo completo.
- Grafo avancado demais.
- Migracao total para outra stack sem necessidade.

## MVP real

Um fluxo completo de caso PF/PJ com BigDataCorp, evidencias, revisao, decisao explicavel e relatorio versionado.

## V2 vendavel

Adicionar dossie, timeline, relatorio premium, modulos por plano e auditoria robusta.

## V2 ambiciosa

Adicionar grafo, watchlists, alertas e comparativo historico.

## Ordem correta

1. Estabilizar V1.
2. Criar camada provider/evidencia.
3. Criar entidades canonicas.
4. Criar dossie.
5. Criar sinais/decisao.
6. Criar report snapshot.
7. Evoluir cockpit.
8. Preparar monitoramento.

## Ganhos competitivos reais

- Confiabilidade e rastreabilidade.
- Dados organizados como evidencia.
- Decisoes defendiveis.
- Relatorios melhores.
- Produto brasileiro especializado.
- Menos retrabalho do analista.
- Melhor experiencia para cliente.
- Base para watchlists e monitoramento futuro.
