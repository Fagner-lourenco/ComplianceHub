Additional resources

Additional features to make your consultations more secure, traceable, and flexible.

This page presents advanced features that complement the use of queries on the BigDataCorp Data Platform.

Here you will learn how to:

Using hashes as an alternative to sending documents directly
Add tags to calls for tracking and organizing flows.
Explore metadata for further analysis of events and integrations.

These features help adapt queries to different security, privacy, and governance requirements.

1️⃣ Hashes as a lookup key

In some scenarios, the client may not feel comfortable directly sending their CPF (Brazilian individual taxpayer registration number) or CNPJ (Brazilian company taxpayer registration number) in a request, whether due to privacy concerns, internal rules, or operational simplicity .

To address these cases, the Data Platform allows the submission of hasheaded document as a lookup key, using the parameter dochashThe hash It must be generated in the standard format. SHA-256, widely adopted in the market.




Example:

JSON
{
    "Datasets": "addresses",
    "q": "dochash{A457CEBA5B8481AFF573336A02A1BB022BD43056F48B022AD0C6320E87EF9EF9}"
}



This feature allows you to perform queries while maintaining compatibility with the APIs, without directly exposing the original document.

2️⃣ Call tags

Tags are custom identifiers used to categorize, organize, and add context to calls made on the Data Platform.

They function as metadata in the format chave: valor, being recorded along with the request information.

All of the Platform's APIs allow the sending of tags.

Among the main uses of tags are:

Identifying workflows (e.g., onboarding, credit analysis)
Differentiation of environments (e.g., testing, production)
Traceability of integrations and internal processes.
Further analysis via the Events API

Examples:

Simple use of tags
JSON
{
  "Datasets": "basic_data",
  "q": "doc{xxxxxxxxxxx}",
  "Tags": {
    "my_key": "my_value",
    "environment": "production"
  }
}



Tags applied to a specific flow
JSON
{
  "Datasets": "registration_data,kyc,occupation_data",
  "q": "doc{xxxxxxxxxxx}",
  "Tags": {
    "host": "HOST_ID",
    "process": "ONBOARDING"
  }
}



📘

These tags can be used as filters in the Events API, allowing you to retrieve calls related to a specific process or business flow.



Rules for using tags

For the request to be processed correctly, the tags must follow the rules below:

You are allowed to send a maximum of 10 tags per request.
Keys and values ​​must be between 1 and 255 characters long .
To the keys can contain only letters of A to Z (without accents) or the character _ (underline).
Values ​​can contain letters, numbers, spaces, and the following special characters: _ * - + , . : ! @ # & /
