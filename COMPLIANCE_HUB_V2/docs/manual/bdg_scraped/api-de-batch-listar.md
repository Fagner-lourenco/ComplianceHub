https://plataforma.bigdatacorp.com.br/lote/listar

This method allows the user to retrieve the list of jobs requested by the user, with detailed data about each of the processes.

Operation

From the provided access token, all currently running batch processing jobs are retrieved, with detailed information (the same as that which can be seen in the GetJobDetails method) about them. The returns are paged according to the page size defined in the call.

Optional Parameter - ShowHistoricalJobs

If you also want to retrieve information about historical jobs, simply include the ShowHistoricalJobs field with the value true in the request. See the body request of the "ListJobs + HistoricalJobs" example below.

Description of main object return fields
Field	Description
TotalNumberOfCurrentJobs	Total number of jobs that are currently running
TotalNumberOfJobs	Total number of jobs already submitted by the user
Jobs	List of objects with detailed information about user jobs
Description of processing detail object return fields
Field	Description
JobId	Unique job identifier
JobName	Job name
JobUserName	User associated with the token
JobDomain	Domain associated with the token
CurrentJobStatus	Current stage (status) of the job
StatusPercentageComplete	Current job stage completeness percentage
TotalRecords	Total number of records sent in the query input
TotalQueriedRecords	Total number of records consulted
TotalParsedRecords	Total number of parsed records
TotalQueryErrors	Total number of query errors occurred
TotalParsingErrors	Total number of parsing errors occurred
FirstQueryItem	String representation of the JSON object referring to the first query performed in the job
QueryAPI	Which API was used to run this job
LastUpdateDate	Last job status update date
Possible Status of the Process

The following table describes the possible values of the CurrentJobStatus field, and the meaning of each of these values.

Value	Description
pending	The job was saved in the database, but no processing was performed on top of it
loading	The job is being loaded for processing in the query queues
loaded	The job was correctly loaded for processing in the query queues, and the user can now command the start of its execution
querying	The job is being queried
parallel_limit_reached	The job is being queried for the maximum number of parallel processes currently available. It is equivalent to querying value
paused	Execution of job queries has been paused until user commands resume
wanted	The job's queries have ended, and it is waiting for the parsing process to generate the output files
parsing	Job query results are being parsed to generate output files
done	The job execution has ended and the final result file sent
cancelled	The job execution was canceled by the user
Status Codes

The following table describes possible return codes and messages.

Code	Message	Description
0	OK	Successful execution
-1	INVALID AUTHENTICATION PARAMETERS	The access token provided in the entry is invalid
-2	BAD REQUEST FORMAT	The sent request is not well formatted
-1200	UNEXPECTED ERROR. PLEASE CONTACT OUR SUPPORT TEAM	An unexpected error has occurred. Get in touch with our service team
