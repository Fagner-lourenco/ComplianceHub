Description

This feature allows you to configure important information for the proper functioning of the Monitoring API, according to the user's needs and preferences. To update any of the authentication settings, simply add the corresponding parameter to the request and enter its new value. To avoid overwriting an already configured field, the field should not be informed in the request.

Input Parameters

The following table describes the input parameters of this method, indicating whether they are required or not.

Field	Mandatory?	Possible Values	Description
AccessToken	Yes	Authentication token in the Data Platform APIs. It can also be informed in the request header.	JWT Token
SftpPrivateKey	No	Private authentication key for SFTP servers.	Access key in OpenSSH format
Authorization Key	No	authorization key	Access key
AuthorizationToken	No	authorization token	Token
OAuthBasicToken	No	Basic OAuth Token	OAuth Token
OAuthPublicKey	No	This field must be used together with the OAuthPrivateKey, if the OAuthBasicToken is not provided	Access key
OAuthPrivateKey	No	OAuth Private Key	Access key
OAuthURLTarget	No	Authentication server access URL	url
OAuthContentType	No	Type of content that should be returned	content type
OAuthGrantType	No	Determines the type of authorization	type of permission
OAuthExpectedTokenFieldName	No	This field is used by the client to point to the field where we will retrieve the access token coming from OAuth (default value is "access_token")	Field name

The AuthorizationToken and OAuthBasicToken fields are not stored in our databases. We use the AWS key persistence service, offering greater security to user credentials.

Return

The following table describes the fields of the return object of this method.

Field	Description
User	Username
ResultStatus	Query status message
