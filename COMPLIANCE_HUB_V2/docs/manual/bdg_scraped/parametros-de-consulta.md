Request Parameters

A guide to the parameters that structure a query on the Platform.

The parameters structure the query and guide the Data Platform on who to locate and what set of information to return .

Here you will learn:

The function of each parameter (q, Datasets, Limit)
How to declare identification keys
The differences between main, alternative, and optional keys.
When and how to combine keys
Complete lists of keys, by type
When to use Limit

These concepts are essential for the following pages to make sense.

1️⃣ Available parameters
Parameter	Description	Type
q	List of search keys for the query, separated by commas.	🛑 Mandatory
Datasets	List of returned datasets, which may include modifiers.	🛑 Mandatory
Limit	Maximum limit of entities returned.	ℹ️ Optional
2️⃣ Regarding the parameter q

The parameter q brings together identification keys sent to locate an entity on the Data Platform.
It functions like the "who am I trying to find" within your search.

Each key follows the format: nome_da_chave{valor}.

Simple example
JSON
"q": "name{Maria Souza}"


❗️
Sending q It is mandatory in all consultations.
🔑 Identification keys

The Data Platform uses three types of identification keys:

Master keys → uniquely identify the entity (1:1)
Alternative keys → can return more than one entity.
Additional keywords → refine the search and increase precision.

Next, we detail each group.

Key Keys

These are identifiers that have a 1:1 relationship with the entity being consulted.

Main Key	Description	Example	API
doc	Individual or Company Identification Number (CPF/CNPJ)	doc{12345678900}	Person or Company
zipcode	Postal Code	zipcode{12345000}	Address
licenseplate	Plate number	licenseplate{ABC1D23}	Vehicle
processnumber	Process number	processnumber{10000000120258260302}	Process
receiptnumber	Invoice Number	receiptnumber{123456789}	Invoice
ean	Barcode Number	ean{12345678890123}	Product
Alternative keys

When the user does not have a primary key, they must submit at least one alternate key. Because they are less specific, it is recommended to submit two or more alternate keys, or combine them with supplementary keys.

Alternative Key	Description	Example	API
name	Entity name	name{josé da silva}	Person or Company
classnumber + classorganization	Registration number with the professional council and name of the organization.	classnumber{123456}, classorganization{CFMSP}	Person
nit	Employee Registration Number (or ID number)	nit{123456789}	Person
rntrc	ANTT Registration Number	rntrc{12345678}	Person or Company
domain	Domain (website URL)	domain{seudominio.com}	Person or Company
name + partialdoc	Name and partial information from the entity's identification document.	name{big corp}, partialdoc{*8768627*}	Person or Company
Additional keys

Complementary keys do not identify on their own, but they help reduce ambiguities, increase accuracy, and complement alternative keys.

Additional keys (click to expand)


💡
Tip:

The more alternative and supplementary keys you send, the greater the chances of reaching the correct entity .

❗️
Attention:

In certain datasets, keys classified as complementary may become mandatory due to specific requirements of the data sources.

When this occurs, the requirement will be explicitly indicated in the "Relevant supplementary keys" section of the respective dataset.

3️⃣ Regarding the parameter Datasets

The parameter Datasets informs which datasets These should be returned by the API. It functions as "what do I want to receive from this entity".

Examples of common datasets:

basic_data
kyc
processes

The complete list of datasets can be found in the specific documentation for each API, in the sidebar menu.

THE Datasets It also allows:

field selection
use of modifiers
Composing multiple datasets in the same call

These features are detailed on the “Customize return” page.

4️⃣ Regarding the parameter Limit

THE Limit controls maximum number of entities Errors returned when the query is not specific enough.

Use when:

if there is no master key
the query could generate multiple occurrences
If you want to limit the number of entities returned


Example
JSON
{
  "Datasets": "basic_data",
  "q": "name{Joao da Silva}, birthdate{10-02-1995}, dateformat{dd-mm-yyyy}",
  "Limit": 5
}
5️⃣ Practical Summary
q → Who am I looking for?
Datasets → what I want to receive
Limit → Maximum number of entities I want to receive when the search key is not 1:1

These three parameters form the basis of all queries on the Data Platform.
