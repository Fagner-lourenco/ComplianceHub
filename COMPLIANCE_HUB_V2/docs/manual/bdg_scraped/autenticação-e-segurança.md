Authentication and security

Ensure secure, scalable, and traceable integrations, preserving data integrity and the reliability of your applications.

Authentication is the starting point for any integration with BigDataCorp products – and security is a fundamental part of this process.

In this section, you will find everything you need to securely access APIs, manage your authentication tokens, and apply best practices for protection and rotation.

Here you will learn how to:

Generate and renew your JWT tokens directly through the BDC Center or via the Token API;
Properly configure the authentication headers in your requests;
Implement security and traceability policies in test and production environments;
Adopt best practices for storing and using credentials .

Our goal is to ensure that your integrations are secure, scalable, and traceable, preserving data integrity and the reliability of your applications.



💡

BigDataCorp's APIs use JWT tokens for authentication. These ensure security, access control, and flexibility in your integrations.



🔐Authentication

If you are not yet a BigDataCorp customer, please contact our sales team at comercial@bigdatacorp.com.br . They will assist you with registration and obtaining your access credentials.

If you are already a customer, access the BDC Center . There, you can change or reset your password at any time.

BigDataCorp API authentication is done using JWT tokens . With your username and password, you can generate as many tokens as you need, ensuring security and flexibility for your business processes.

Tokens can be generated directly in the BDC Center or via the Token API . It's important to remember that all tokens have an expiration date—there are no permanent tokens.

The Token API is available at no additional cost, allowing your company to implement a token rotation policy that better suits your needs. To learn more about recommendations and best practices, see the Security section.



⚠️

Attention: Your access token is unique and should not be shared. You will be responsible for the cost of all queries made using your access token.



Authentication framework

All calls to the BigDataCorp Data Platform follow the same authentication pattern.

Endpoint base:

https://plataforma.bigdatacorp.com.br/

Required headers:

AccessToken: SEU-ACCESS-TOKEN
TokenId: SEU-TOKENID
Content-Type: application/json
Accept: application/json

These tokens can be obtained from the BDC Center or via the Tokens API .



💡
Tip

The authentication format is the same for all APIs on the BigDataCorp Data Platform . Simply adjust the endpoint according to the desired entity . For example:

• .../pessoas → calls to datasets of People
• .../empresas → calls to datasets of Companies



🛡️Security


Usernames and passwords

User and password management is handled by the BDC Center . We recommend that each user have their own account and that credentials are never shared .

When integrating our APIs into your systems, use dedicated service users, always associated with valid corporate email addresses.

In production environments, generate and use tokens linked to these service users, ensuring traceability and security in the use of APIs.



Token Rotation

Although the Token API allows tokens to be generated with a validity of up to 10 years (87,600 hours), we suggest rotating tokens daily . We do not recommend rotating tokens more than once within the same hour.

Consult the Tokens API documentation — it offers resources that allow you to implement the token rotation policy best suited to your organization.



Tokens and web applications

Use tokens only in applications that operate on the backend . They function as your authentication for BigDataCorp APIs and can be exposed or compromised if used directly on the application's frontend .



Access restriction

If your architecture requires additional security layers, you can request the configuration of access restrictions to the Data Platform APIs, limiting calls to specific domains (CORS) or authorized IP ranges .

To enable any of these settings, contact your Customer Success representative, informing them of the domain that needs to be authorized (CORS) or the allowed IP ranges .



Request limit

For security and platform stability reasons, the APIs have a rate limit of 5,000 queries every 5 minutes (per IP address) . Requests should be distributed throughout this interval, avoiding call spikes in short periods.

If the limit is exceeded, the API will return an error. HTTP 429 — Too Many Requests.
