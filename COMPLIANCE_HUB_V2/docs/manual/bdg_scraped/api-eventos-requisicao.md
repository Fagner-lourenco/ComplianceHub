Description

The Events API allows you to retrieve log and audit records of queries executed on the Data Platform. Each query generates a record that includes information such as execution time, source IP, parameters used, and the returned result. All query records are stored indefinitely and are available for access via the API for at least 3 months. To obtain older records, please contact us .

Request Parameters

The request must contain a JSON body with the search parameters. It is mandatory to provide one of the following parameters: QueryId, q or Tags.

Parameter	Type	Mandatory	Description
QueryId	string	Conditional	Unique identifier of the query to be retrieved.
q	string	Conditional	Search string to filter events based on the original query parameters. Ex: doc{12345678901}.
Tags	object	Conditional	Object with chave:valor to search queries by custom tags. The search will return events that contain all the tags provided.
QueryDate	string	Optional	Reference date for the query in the format specified in DateFormat. Optimizes search when used with QueryId or filters results when used with q.
DateFormat	string	Optional	Date format reported in QueryDate. Ex: yyyy-MM-dd. Mandatory if QueryDate is informed.
PageSize	integer	Optional	Sets the number of records per page. Used only with the parameter q. Default value: 50.
Cursor	string	Optional	Pointer to the next page of results, obtained from a previous response. Used for pagination in conjunction with q.

Note: We strongly recommend that all QueryIDs be stored, as this is the strongest link between queries and log and audit records. By default, a user has access to all event records associated with their organization.

Examples

Retrieve the result of a specific query faster, with a reference date:

{
    "QueryId": "[seu query id]",
    "QueryDate": "2019-07-25",
    "DateFormat": "yyyy-MM-dd"
}

Retrieve all queries that use specific tags:

{
    "Tags": {
        "Key1": "Value1",
        "Key2": "Value2"
    }
}
Retrieve all queries made against a specific CPF, in sets of 10:
{
    "q": "doc{xxxxxxxxxxx}",
    "PageSize": 10
}
Perform pagination by results:
{
    "q": "doc{xxxxxxxxxxx}",
    "PageSize": 10,
    "Cursor": "[valor de cursor retornado pela chamada anterior]"
}
