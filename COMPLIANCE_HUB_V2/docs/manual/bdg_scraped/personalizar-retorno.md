Customize return

Personalize the results of your queries by selecting fields, applying filters, controlling sorting, paginating results, limiting results, and using a reference date .

The resources on this page allow you to refine the response from the Data Platform APIs, delivering only the information necessary for your workflow.

Here you will learn how to:

Select specific fields from a dataset .
Filter returned records
Sorting lists of data
Paginate results
Limit the number of results
Use reference date

These features help reduce the volume of data transferred, improve the performance of integrations, and make the responses more aligned with your needs.

1️⃣ Select fields from a dataset

In some scenarios, it is not necessary to receive all available fields in a dataset.
For this, it is possible to provide this information at the time of the call. which fields should be returned, directly in the parameter Datasets.

Field selection is done by specifying the dataset name followed by the desired fields in curly braces, separated by commas ({campo1, campo2}).

Examples:

Selecting fields in a dataset
JSON
Body:
{
    "Datasets": "basic_data{name, birthdate, taxidstatus, mothername}",
    "q": "doc{12345678900}"
}
Multiple datasets with selection in only one of them.

When the call includes more than one dataset, the field selection can be applied only to the desired one:

JSON
Body:
{
    "Datasets": "basic_data{name, birthdate, taxidstatus, mothername}, kyc, processes",
    "q": "doc{12345678900}"
}
❗️
In queries to the On-demand API, it is not possible to select fields to be returned.
2️⃣ Filter returned results

Some datasets allow you to filter the returned results, selecting only information that meets specific criteria.

The filtering rules follow the principles below:

There is no distinction between uppercase and lowercase letters.
It is possible to filter by multiple values ​​in the same field.
It is possible to apply filters to more than one field simultaneously.
Simple filter
JSON
{
    "Datasets": "processes.filter(party_type=defendant)",
    "q": "doc{12345678900}"
}
Filter with multiple values
JSON
{
    "Datasets": "phones.filter(areacode=[21,11])",
    "q": "doc{12345678900}"
}
Filters across multiple fields
JSON
{
    "Datasets": "emails.filter(Type=work,validationstatus=VALID)",
    "q": "doc{12345678900}"
}
📘
To find out if a specific dataset supports filters, consult its documentation.
3️⃣ Sort results lists

Several datasets return lists of results, such as in the cases of contacts (phone numbers, addresses, and emails) and relationships.

By default, these lists are returned following an internal prioritization order, which considers relevance criteria according to the data context.

If it becomes necessary to change this sorting, it is possible to sort the results in a certain way. growing(ascendingor decreasing(descending) by any field available in the dataset.

Sorting results
JSON
{
    "Datasets": "emails.order(EmailTotalPassages=ascending)",
    "q": "doc{12345678900}"
}
4️⃣ Paginate results

Some datasets return extensive lists of recordsWhen more results are available, the return will include the field. Nextcontaining the Next page identifier.

To retrieve the next records, use the parameter. .next(x) in the field Datasets, right after the dataset name, indicating the value returned in Next and maintaining the same parameter q from the original query.



Retrieve next page
JSON
{
    "Datasets": "media_profile_and_exposure.next(AoJw14Wy0ZYDP0No...)",
    "q": "doc{12345678900}"
}
📘
Important:

Pagination does not incur any additional cost, and subsequent pages remain available for up to 3 hours after the initial search.

5️⃣ Limit the number of results

Some datasets may return a large number of records for the same entity, such as Awards and Certifications, Contacts, or Relationships.

In these scenarios, it may be beneficial to define a maximum limit on the number of results returned, ensuring faster, more organized responses that are aligned with your business workflow.

To do this, use the parameter. .limit(x) in the field Datasets, right after the name of the dataset to which you want to apply this customization.



Limiting the number of results
JSON
{
    "Datasets": "awards_and_certifications.limit(10)",
    "q": "doc{12345678900}"
}
6️⃣ Use reference dates

It is possible to conduct queries considering a specific date as a reference, returning a snapshot of the data at that moment. Sending the keys is mandatory for this. referencedate{DATA} and dateformat{FORMATO} within the parameter q.

This feature is useful for historical analysis and backtesting, without requiring any additional request to the BigDataCorp team.

Examples of calls with a reference date.
JSON
{
    "Datasets": "addresses",
    "q": "doc{12345678900},referencedate{03-09-2018},dateformat{dd-MM-yyyy}"
}


JSON
{
    "Datasets": "political_involvement",
    "q": "doc{12345678900},referencedate{12-2020},dateformat{MM-yyyy}"
}


🚧
If the date submitted is invalid, the current date will be considered as the reference.
7️⃣ Practical Summary
Field selection → defines which information will be returned.
Filters → define which records should appear in the return.
Sorting → defines the order in which the data is presented.
Pagination → allows you to navigate through large volumes of records.
Limitation → defines the maximum number of records that should appear in the return.
Reference date → returns a historical snapshot of the data.

These resources can be combined to create leaner, more organized responses that are aligned with your workflow.
